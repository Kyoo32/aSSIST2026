### 1. GROUP BY — 엑셀의 "피벗 테이블"

- `GROUP BY`는 같은 값을 가진 행들을 묶어서 집계하는 문법이다.
- 엑셀에서 피벗 테이블로 "부서별 매출 합계"를 보는 것과 같은 개념이다.
- 보통 `COUNT`, `SUM`, `AVG`, `MAX`, `MIN` 같은 집계 함수와 함께 쓴다.

핵심 흐름은 보통 이렇다.

1. `WHERE`로 행을 먼저 필터링한다.
2. `GROUP BY`로 묶는다.
3. 집계 함수를 계산한다.
4. 필요하면 `HAVING`으로 그룹 결과를 다시 필터링한다.

> 한 줄 정리: `GROUP BY`는 여러 행을 요약해서 그룹별 통계를 만드는 도구다.

### 2. JOIN — 테이블 합치기

- `JOIN`은 공통 컬럼을 기준으로 여러 테이블을 연결하는 문법이다.
- 엑셀 비유로는 `VLOOKUP`처럼 다른 표의 정보를 이어 붙이는 작업에 가깝다.

대표 유형은 다음과 같다.

- **INNER JOIN**: 양쪽에 모두 매칭되는 행만 반환
- **LEFT JOIN**: 왼쪽 테이블의 모든 행을 유지하고, 오른쪽은 매칭되는 값만 붙임
- **RIGHT JOIN**: 오른쪽 테이블 기준으로 유지
- **FULL OUTER JOIN**: 양쪽 전체를 모두 포함
- **SELF JOIN**: 같은 테이블을 자기 자신과 조인

헷갈리기 쉬운 포인트:

- `LEFT JOIN` 뒤에 개수를 셀 때는 `COUNT(*)`보다 **오른쪽 컬럼 기준 `COUNT(오른쪽_컬럼)`** 을 의식해야 한다.
- 그래야 매칭이 없는 행을 0건으로 올바르게 볼 수 있다.

> 한 줄 정리: `JOIN`은 테이블을 합쳐서, 한 테이블에 없는 정보를 다른 테이블에서 끌어오는 도구다.

### 3. 서브쿼리 — 쿼리 안의 쿼리

- 서브쿼리는 SQL 문 안에 들어 있는 또 다른 SQL 문이다.
- 위치에 따라 역할이 달라진다.

대표 유형은 다음과 같다.

- **WHERE절 서브쿼리**: 안쪽 쿼리 결과를 조건으로 사용
- **FROM절 서브쿼리**: 서브쿼리 결과를 임시 테이블처럼 사용
- **EXISTS 서브쿼리**: 결과가 존재하는지만 확인

예를 들어:

- 평균보다 급여가 높은 의사 찾기
- 진료과별 최고 급여 의사 찾기
- 진단 기록이 있는 방문만 찾기

> 한 줄 정리: 서브쿼리는 "먼저 작은 답을 구한 뒤, 그 답을 바깥 쿼리에서 활용하는 방식"이다.

### 4. CTE — "단계별 레시피"

- CTE(Common Table Expression)는 `WITH`로 시작하는 이름 있는 중간 결과다.
- 복잡한 쿼리를 한 번에 쓰지 않고, 단계별로 나눠 읽기 쉽게 만드는 방식이다.

CTE의 장점:

- 이름을 붙여서 가독성이 좋아진다.
- 여러 번 재사용할 수 있다.
- 각 단계를 따로 실행해보기 쉬워 디버깅이 편하다.

강의 표현대로 보면, CTE는 **요리 레시피**처럼 읽으면 된다.

1. 1단계 재료 준비
2. 2단계 중간 결과 만들기
3. 마지막에 최종 결과 조합

> 한 줄 정리: CTE는 긴 쿼리를 "한 번에 읽는 코드"에서 "단계별로 읽는 코드"로 바꿔준다.

### 5. 윈도우 함수 — "모든 행에 메모 추가하기"

- 윈도우 함수는 원래 행 수를 줄이지 않으면서, 각 행 옆에 추가 정보를 붙이는 함수다.
- 비유하면 "모든 행에 메모를 하나씩 더 써 넣는 것"에 가깝다.

`GROUP BY`와의 차이는 아주 중요하다.

- **`GROUP BY`**: 10행을 3행처럼 요약할 수 있다.
- **윈도우 함수**: 10행은 그대로 두고, 순위/누적합/이전값 같은 새 컬럼을 추가한다.

대표 예시:

- `ROW_NUMBER()`: 순번
- `RANK()`: 순위
- `LAG()`: 이전 행 값 비교
- `SUM() OVER (...)`: 누적합 또는 그룹 내 합계

판별 기준은 간단하다.

- 결과 행 수가 **줄어들어야** 하면 `GROUP BY`
- 결과 행 수가 **유지되어야** 하면 윈도우 함수

> 한 줄 정리: 윈도우 함수는 "행을 요약하지 않고, 각 행에 비교 정보나 순위를 덧붙이는 도구"다.
