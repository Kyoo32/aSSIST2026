## Embedding And Vector Basics

### 1. 임베딩이란? — "단어의 좌표"

- 임베딩은 텍스트를 숫자 벡터로 바꾸는 과정이다.
- 쉽게 말하면, 문장이나 단어를 **의미 공간의 좌표**로 옮기는 것이다.

예를 들면:

- `"두통"` -> 가까운 좌표
- `"머리가 아파요"` -> `"두통"`과 가까운 좌표
- `"오늘 날씨"` -> 멀리 떨어진 좌표

즉, **의미가 비슷한 텍스트는 벡터도 가까워진다**.

> 한 줄 정리: 임베딩은 텍스트를 "검색 가능한 의미 좌표"로 바꾸는 작업이다.

### 2. 코사인 유사도 — "화살표 방향이 비슷한가?"

- 코사인 유사도는 두 벡터가 얼마나 비슷한 방향을 가리키는지 보는 방법이다.
- 임베딩끼리 의미 유사성을 비교할 때 가장 자주 쓰인다.

해석은 보통 이렇게 기억하면 된다.

- **1에 가까움**: 거의 같은 방향 -> 매우 유사
- **0에 가까움**: 직각에 가까움 -> 관련 적음
- **-1에 가까움**: 반대 방향 -> 반대 의미

중요한 점은, 크기보다 **방향**을 더 본다는 것이다.

> 한 줄 정리: 코사인 유사도는 "두 의미 벡터가 같은 방향을 보고 있는가?"를 측정한다.

### 3. 임베딩 모델 비교

강의에서 소개된 주요 모델은 다음과 같다.

| 모델 | 차원 | 비용 | 특징 |
| --- | --- | --- | --- |
| `text-embedding-3-small` | 1536 | 저렴 | 빠르고 경제적, 대부분의 경우 충분 |
| `text-embedding-3-large` | 3072 | 보통 | 더 높은 정확도, 정밀 검색에 유리 |
| `BAAI/bge-m3` | 1024 | 무료 | 다국어 포함, 최신 오픈 임베딩 |
| `jhgan/ko-sroberta-multitask` | 768 | 무료 | 한국어 특화 |
| `paraphrase-multilingual-MiniLM-L12-v2` | 384 | 무료 | 다국어, 경량 |

선택 기준은 보통 이렇다.

- **프로토타입/학습**: `text-embedding-3-small`
- **정확도 우선 프로덕션**: `text-embedding-3-large`
- **비용 제한/로컬 실행**: `BAAI/bge-m3` 또는 `jhgan/ko-sroberta-multitask`

### 4. 2D 시각화 개념 — t-SNE

- 임베딩 벡터는 보통 수백~수천 차원이어서 사람이 직접 볼 수 없다.
- `t-SNE`는 이런 고차원 벡터를 2차원으로 줄여서 시각화할 때 자주 쓰는 방법이다.

핵심 감각은 단순하다.

- 원래 고차원에서 가까운 점들은
- 2차원으로 줄인 뒤에도 가깝게 보이도록 배치하려는 것이다

그래서 `두통`, `머리아파` 같은 비슷한 표현은 시각화에서도 가까이 모이고,
`날씨`처럼 관련 없는 표현은 멀리 떨어져 보이게 된다.

> 한 줄 정리: t-SNE는 "보이지 않는 고차원 의미 공간"을 사람이 눈으로 볼 수 있게 펼쳐주는 도구다.
